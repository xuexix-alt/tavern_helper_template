# 模块5：交互式HTML状态栏生成指令（MVU Zod版本）

## 模块定义

本模块是 MVU Zod 系统的最终展示层，负责生成一个包含 `<head>` 和 `<body>` 的 HTML 片段。该片段将包含 CSS 样式、HTML 结构和 JavaScript 逻辑，用于创建一个可交互、能动态展示前面模块定义的 MVU 变量的状态栏。

---

## 核心要求

### 1. 可用库

状态栏可以直接使用以下全局库，无需额外导入：
- **jQuery** - DOM 操作
- **jQuery UI** - UI 组件
- **lodash** (`_`) - 工具函数
- **Zod** (`z`) - 类型验证
- **toastr** - 提示消息

### 2. CSS 样式

- 根据角色设定或氛围来编写 `<style>` 内的样式
- 若无特殊要求，状态栏应设计为卡片式结构，且不设置整体的背景颜色

### 3. HTML 结构 (`<body>`)

- 基于提供的模板，在 `<div class="card-body">` 内，根据需要展示的变量自定义内容
- 为每个需要展示的变量创建对应的 HTML 元素
- 为需要动态更新的元素分配唯一的 `id`，使用 jQuery 选择器访问

### 4. JavaScript 逻辑 (`<script type="module">`)

#### 主要任务

编写 `populateCharacterData()` 函数来填充状态栏数据。

#### 数据源

**获取方式**：
```javascript
const all_variables = getAllVariables();
const characterData = all_variables.stat_data;
```

状态栏中展示的变量及其路径，应严格参考模块1中定义的变量初始化数据。

**重要说明**：MVU Zod 框架会自动处理变量的类型验证和边界控制，状态栏只需关注数据的展示。

#### 实现步骤

1. 使用 `getAllVariables()` 获取全局变量
2. 从 `all_variables.stat_data` 中提取角色数据
3. 使用 `_.get(all_variables, 'stat_data.变量路径', '默认值')` 安全地获取变量值
4. 使用 jQuery（如 `$('#element-id').text(value)`）更新 DOM 元素
5. 对于进度条，需要计算百分比并使用 `$('#bar-id').css('width', percentage + '%')`

#### 初始化流程

```javascript
async function init() {
    /* 等待 MVU 系统初始化完成 */
    await waitGlobalInitialized('Mvu');

    /* 填充数据 */
    populateCharacterData();

    /* 绑定事件（如需要） */
    $('.section-header').on('click', function () {
        toggleSection($(this));
    });
}

/* 使用 errorCatched 包装并在 jQuery ready 时执行 */
$(errorCatched(init));
```

#### 约束

- **必须使用 jQuery** 进行 DOM 操作，而非原生 JavaScript
- **必须使用 `_.get()`** 安全获取嵌套变量，而非直接访问
- **必须使用 `/*注释*/` 格式**，禁止使用 `//注释`，否则可能渲染失败
- **必须使用 `errorCatched` 包装**主初始化函数
- **必须在 `$(() => {})` 中**执行初始化代码
- **必须等待** `waitGlobalInitialized('Mvu')` 完成后再访问数据

### 5. 最终输出

输出内容应包含：
1. `<head>` 标签（包含 `<style>` 和 `<script type="module">`）
2. `<body>` 标签（包含 HTML 结构）

**不需要**包含 `<!DOCTYPE html>`、`<html>` 或 `</html>` 标签。

---

## 输出要求

### 1. 核心工作

基于以下模板进行定制：

1. 根据模块1中定义的变量，修改 HTML 的 `<body>` 部分以定义要展示的属性
2. 在 JavaScript 的 `populateCharacterData` 函数中编写逻辑以填充这些属性
3. 确保展示的变量与模块2中定义的更新规则相对应
4. 状态栏将实时反映通过模块3（JSON Patch）更新的变量值

### 2. 变量映射

- 所有显示的变量必须来自模块1的初始化定义
- 变量路径必须与模块1中的 YAML 结构完全一致
- 使用点分隔符路径（如 `'凌若雪.好感度'`）配合 `_.get()` 获取值

### 3. 系统集成

- **变量来源**：显示的变量必须来自模块1的初始化定义
- **数据更新**：变量值的更新遵循模块2的规则，通过模块3的 JSON Patch 指令执行
- **类型安全**：Zod Schema 自动处理类型转换和边界控制
- **用户体验**：为用户提供直观的 MVU 系统运行状态反馈

### 4. 调试建议

若状态栏无法正常显示，请：
1. 检查浏览器控制台的错误信息
2. 确保变量路径与模块1的定义一致
3. 确认使用了正确的 jQuery 语法
4. 检查是否等待了 `waitGlobalInitialized('Mvu')`

        /* 更新进度条辅助函数 */
        function updateProgressBar(barId, valueId, rawValue, min = 0, max = 100, color = '#667eea') {
            const numericValue = parseFloat(rawValue);
            if (!isNaN(numericValue)) {
                const clampedValue = _.clamp(numericValue, min, max);
                const percentage = max - min === 0 ? 0 : ((clampedValue - min) / (max - min)) * 100;

                $(`#${valueId}`).text(clampedValue);
                $(`#${barId}`).css({
                    width: `${percentage}%`,
                    backgroundColor: color
                });
            } else {
                $(`#${valueId}`).text(rawValue);
            }
        }

        /* 填充状态栏数据 */
        function populateCharacterData() {
            try {
                const all_variables = getAllVariables();
                const characterData = all_variables.stat_data;

                if (!characterData) {
                    $('#status-card').html("<div class='error-tip'>无法加载状态数据</div>");
                    return;
                }

                /* 世界状态 */
                $('#current-time').text(_.get(characterData, '世界.当前时间', 'N/A'));
                $('#current-location').text(_.get(characterData, '世界.当前地点', 'N/A'));

                /* 任务列表 */
                const taskList = _.get(characterData, '任务列表', {});
                let taskHtml = '';
                _.forEach(taskList, (task, taskKey) => {
                    const taskType = task['任务类型'] || '未知';
                    const taskDesc = task['任务描述'] || '无描述';
                    const taskName = task['任务名'] || taskKey;
                    const typeClass = taskType === '主线' ? 'task-type-main' : 'task-type-side';
                    taskHtml += `<article class="task-item"><div class="task-header"><span class="task-type ${typeClass}">${taskType}</span><span class="task-name">${taskName}</span></div><p class="task-desc">${taskDesc}</p></article>`;
                });
                $('#task-list').html(taskHtml || '<div class="empty-tip">暂无任务</div>');

                /* 江湖传记 */
                const achievements = _.get(characterData, '江湖传记', []);
                if (Array.isArray(achievements) && achievements.length > 0) {
                    let achHtml = '';
                    achievements.forEach((ach, index) => {
                        achHtml += `<div class="achievement-item" data-index="${index + 1}">${ach}</div>`;
                    });
                    $('#achievements-list').html(achHtml);
                } else {
                    $('#achievements-list').html('<div class="empty-tip">尚无事迹</div>');
                }

                /* 凌若雪 */
                const lingRuoxueInTeam = _.get(characterData, '凌若雪.是否相遇');
                if (lingRuoxueInTeam === true || lingRuoxueInTeam === 'true') {
                    $('#tab-lingruoxue').show();
                    updateProgressBar(
                        'lingruoxue-affection-bar',
                        'lingruoxue-affection',
                        _.get(characterData, '凌若雪.好感度', 0),
                        0, 100, '#ff6b9d'
                    );
                    $('#lingruoxue-relationship').text(_.get(characterData, '凌若雪.关系状态', 'N/A'));
                    const lrxInTeam = _.get(characterData, '凌若雪.是否相遇');
                    $('#lingruoxue-inteam').text(lrxInTeam === true || lrxInTeam === 'true' ? '已入队' : '未入队');
                    $('#lingruoxue-outfit').text(_.get(characterData, '凌若雪.当前着装', 'N/A'));
                    $('#lingruoxue-pose').text(_.get(characterData, '凌若雪.当前姿势', 'N/A'));
                    $('#lingruoxue-thoughts').text(_.get(characterData, '凌若雪.当前想法', 'N/A'));
                } else {
                    $('#tab-lingruoxue').hide();
                    $('#tab-content-lingRuoxue').removeClass('active');
                }

                /* 慕容芷 */
                const murongZhiInTeam = _.get(characterData, '慕容芷.是否相遇');
                if (murongZhiInTeam === true || murongZhiInTeam === 'true') {
                    $('#tab-murongzhi').show();
                    updateProgressBar(
                        'murongzhi-affection-bar',
                        'murongzhi-affection',
                        _.get(characterData, '慕容芷.好感度', 0),
                        0, 100, '#4ecdc4'
                    );
                    $('#murongzhi-relationship').text(_.get(characterData, '慕容芷.关系状态', 'N/A'));
                    const mrzInTeam = _.get(characterData, '慕容芷.是否相遇');
                    $('#murongzhi-inteam').text(mrzInTeam === true || mrzInTeam === 'true' ? '已入队' : '未入队');
                    $('#murongzhi-outfit').text(_.get(characterData, '慕容芷.当前着装', 'N/A'));
                    $('#murongzhi-pose').text(_.get(characterData, '慕容芷.当前姿势', 'N/A'));
                    $('#murongzhi-thoughts').text(_.get(characterData, '慕容芷.当前想法', 'N/A'));
                } else {
                    $('#tab-murongzhi').hide();
                    $('#tab-content-murongZhi').removeClass('active');
                }

                /* 蓝灵儿 */
                const lanLingerInTeam = _.get(characterData, '蓝灵儿.是否相遇');
                if (lanLingerInTeam === true || lanLingerInTeam === 'true') {
                    $('#tab-lanlinger').show();
                    updateProgressBar(
                        'lanlinger-affection-bar',
                        'lanlinger-affection',
                        _.get(characterData, '蓝灵儿.好感度', 0),
                        0, 100, '#a29bfe'
                    );

                    $('#tab-lanlinger').hide();
                    $('#tab-content-lanLinger').removeClass('active');
                }

                /* 检查激活的标签是否被隐藏 */
                const $activeTab = $('.tab-button.active');
                if ($activeTab.css('display') === 'none') {
                    switchTab('main', $('[data-tab="main"]'));
                }
            } catch (error) {
                $('#status-card').html(`<div class='error-tip'>状态栏加载出错: ${error.message}</div>`);
            }
        }

        /* 切换标签 */
        function switchTab(tabName, $tabElement) {
            $('.tab-button').removeClass('active');
            $('.tab-panel').removeClass('active');

            if ($tabElement) {
                $tabElement.addClass('active');
            } else {
                $(`[data-tab="${tabName}"]`).addClass('active');
            }

            $(`#tab-content-${tabName}`).addClass('active');
        }

        /* 初始化 */
        async function init() {
            await waitGlobalInitialized('Mvu');
            populateCharacterData();

            /* 绑定标签切换事件 */
            $('.tab-button').on('click', function () {
                const tabName = $(this).data('tab');
                switchTab(tabName, $(this));
            });
        }

        $(errorCatched(init));
    </script>

## MVU Zod 版本说明

本版本与旧版 MVU Beta 的主要区别：

1. **数据获取**：使用 `getAllVariables()` 而非 `getChatMessages()`
2. **DOM 操作**：使用 jQuery 而非原生 JavaScript
3. **安全访问**：使用 `_.get()` 而非自定义 `SafeGetValue`
4. **初始化**：需要 `await waitGlobalInitialized('Mvu')`
5. **错误处理**：使用 `errorCatched` 包装函数
6. **注释规则**：只能使用 `/**/` 格式
7. **HTML 结构**：只需 `<head>` 和 `<body>`，无需完整文档声明
8. **类型安全**：Zod Schema 自动处理类型转换和边界控制
